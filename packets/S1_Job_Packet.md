# Project Packet — LawnCare Lite

**Assigned:** July 2026
**Project Type:** New Build
**Status:** Ready for Development

---

## 1. Project Overview

Build a web application for GreenThumb Lawn Care, a small lawn care company. The app will replace their current paper-based scheduling system.

## 2. Business Context

GreenThumb Lawn Care has been operating for 4 years. They currently manage everything on paper — customer info on clipboards, schedules on a whiteboard, job details on notepads. They've grown to the point where this is no longer sustainable. The owner wants a simple digital system.

Budget is limited. They need something that works reliably, not something that's overbuilt.

## 3. Users

The application will have the following user types:

- **Office Manager** (1 person) — manages customers, schedules jobs, views reports
- **Field Technicians** (2 people) — view their daily schedule, update job status

All users access the application through a web browser.

## 4. Functional Requirements

### 4.1 Customer Management
- Add, edit, and view customer information
- Search customers

### 4.2 Appointment Scheduling
- Schedule appointments for customers
- Assign appointments to technicians

### 4.3 Job Completion
- Technicians can mark jobs as complete in the field

### 4.4 Search
- Search for customers

### 4.5 Daily Schedule
- Print daily schedule

## 5. Non-Functional Requirements

- **Ease of use:** The application must be easy to use. The office manager and technicians are not highly technical.
- **Responsive UI:** Must work on desktop and mobile browsers. Technicians will use their phones in the field.
- **Reliability:** The application must be reliable. This is a small business that depends on it daily.

## 6. Technical Constraints

- Web application
- Must work in desktop browsers and mobile browsers (phones and tablets)

## 7. Security

- Login required for all users
- No payment data or medical data is involved

## 8. Deliverables

- Architecture documentation
- Source code
- Tests
- README with setup instructions
- ASSUMPTIONS.md documenting any assumptions made
- Build instructions

## 9. Business Operations

GreenThumb is a small but real business. The owner, Maria, has been running it for 4 years and plans to keep growing. She mentioned she might sell the business in a few years if the right offer comes along.

The two technicians, Jake and Sam, have been with the company since the beginning. Maria trusts them but wants to make sure the system doesn't create unnecessary friction or expose information that shouldn't be shared.
